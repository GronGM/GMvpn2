# GMvpn v1.1 roadmap

`v1.1.0` is not approved and no `v1.1.0-rc.1` tag exists. This roadmap
tracks product improvements after the GitHub MVP/internal
`android-v1.0.0` pre-release.

Current public tester build:

- release: `android-v1.0.0`;
- APK: `GMvpn-android-v1.0.0.apk`;
- release type: GitHub pre-release, MVP/internal;
- Google Play: not published.

Known limitations remain:

- UDP: `pass_limited`;
- DNS: pass for the tested device/network;
- IPv6: `not_tested`.

## Candidate scope

- Premium dark Material 3 UI.
- Centralized design tokens.
- Reusable Compose components.
- Bottom navigation shell with Home / Profiles / Import / Settings.
- Trust-first home screen.
- Dedicated profile management screen.
- Dedicated subscription/manual import screen.
- Dedicated privacy settings screen.
- Safer masked manual profile and subscription entry.
- Better profile management polish.
- Clearer import preview and duplicate/skip summary.
- Redacted diagnostics UX.
- Accessibility and localization cleanup.

## v5 reference baseline decision

v5 reference previews are accepted as the baseline for live UI mapping.

Accepted for mapping:

- layout direction;
- density;
- cards;
- buttons;
- profile rows;
- bottom navigation;
- Home / Profiles / Import / Privacy reference screen structure.

Icons are accepted as temporary/reference-quality and can be improved in
a later dedicated icon fidelity pass. They are not a blocker for the
first live layout/component mapping pass.

This decision only closes the preview-only stage. It does not:

- move preview UI into live screens;
- create or approve `v1.1.0-rc.1`;
- create a tag;
- create a GitHub Release;
- publish Google Play.

## Live UI implementation order

1. Theme/tokens live integration.
2. Home live mapping.
3. Profiles live mapping.
4. Import live mapping.
5. Privacy/settings live mapping.
6. Icon fidelity pass.

Each step must preserve privacy-safe labels and avoid endpoint, URI,
UUID, password, IP, host, port, base64 payload, subscription URL, or
query-like secret labels in ordinary UI.

Stage 1 progress:

- Branch `codex/p2-live-home-premium-ui` starts the first live mapping
  step.
- Theme/tokens and Home live mapping are implemented together so the
  accepted v5 visual system is visible in the real Home flow.
- Profiles, Import and Settings remain future live mapping steps. They
  may inherit shared tokens/components, but their product flows are not
  redesigned in this step.
- The stage preserves existing tunnel/profile/import/diagnostics
  business logic and does not change release metadata.
- A later icon fidelity pass is allowed before any `v1.1.0-rc.1`.

Stage 2 progress:

- The same branch starts live Profiles mapping after the Home baseline.
- Profiles rows now follow the v5 direction with standalone premium rows,
  active/inactive state, safe profile labels, protocol, latency, active
  badge, outline select action and kebab/details action.
- Clear library remains a muted destructive action, not the primary CTA.
- Details, rename, delete confirmation, active selection and
  active-profile reset behavior remain on the existing code paths.
- Import and Settings are still future live mapping stages.
- Unavailable profile styling remains a future option until the runtime
  model exposes a real unavailable/disabled profile state.

Stage 2.5 progress:

- Home/main menu is accepted for the current stage.
- No additional Home/Menu visual pass is planned before Import mapping.
- Active profile remains on Home, but the redundant heading was removed.
- Saved profiles preview was removed from Home; profile browsing belongs
  to the Profiles tab.

Stage 3 progress:

- Import live mapping is complete for the current stage.
- The result is a compact privacy-first Import flow with masked
  subscription/manual inputs, safe preview, redacted import messages and
  no raw URL/URI, endpoint, UUID, password, base64 payload or subscription
  URL in ordinary UI.
- Settings/Privacy mapping followed as Stage 4; icon fidelity followed as
  Stage 5.

Stage 4 progress:

- Settings/Privacy live mapping is complete for the current stage.
- The screen now uses the same compact premium card system as
  Home/Profiles/Import.
- Routing, privacy-first, system kill switch and diagnostics cards keep
  existing actions and do not expose endpoint/profile details.
- Full real VPN smoke remains future work before any tester RC.

Stage 5 progress:

- Icon fidelity pass is complete for the current live UI stage.
- `GmLineIcon` remains the single live Compose/Canvas icon source.
- Bottom navigation and card/action icons now use a more consistent
  2dp line style with rounded caps and joins.
- PNG reference tiles remain local in ignored `.local/design-assets` and
  are not committed.
- Business logic, parser behavior, storage, diagnostics redaction and
  release metadata are unchanged.

## Release criteria

Before any `v1.1.0-rc.1`:

- code changes are complete and reviewed;
- `values` / `values-ru` string parity passes;
- unit tests pass;
- `lintDebug` passes;
- `assembleDebug` passes;
- physical smoke passes on a real Android device;
- no ordinary UI surface shows endpoint, URI, UUID, password, IP, host,
  port, base64 payload, subscription URL, or query-like secret labels;
- no `.local/`, APK/AAB, screenshots with private profiles, raw logs, raw
  evidence, or private configs are committed;
- release notes disclose remaining UDP/IPv6 limitations if still open;
- signed workflow and artifact verification are run only after explicit
  release approval.

- Google Play publication.
