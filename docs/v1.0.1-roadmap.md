# GMvpn Android v1.0.1 roadmap

`v1.0.1` is not approved and is not created. This roadmap is a triage
workspace for feedback after the GitHub MVP/internal `android-v1.0.0`
pre-release.

Current public tester build:

- release: `android-v1.1.0-rc.1`;
- APK: `GMvpn2-android-v1.1.0-rc.1-signed.apk`;
- release type: GitHub pre-release, tester APK;
- AAB asset: not uploaded for ordinary testers;
- Google Play: not published;
- known limitations: diagnostics/TalkBack `pass_limited`, UDP
  `pass_limited`, IPv6 `not_tested`.

Historical `v1.0.1` triage baseline:

- release: `android-v1.0.0`;
- APK: `GMvpn-android-v1.0.0.apk`;
- release type: GitHub pre-release, MVP/internal;
- Google Play: not published;
- known limitations: UDP `pass_limited`, DNS `pass` for the tested
  device/network, IPv6 `not_tested`.

## Candidate fixes from tester feedback

Use this section only for issues confirmed from tester reports. Do not
create `v1.0.1` for documentation-only changes.

Current state:

- New tester issues: 0.
- Confirmed `v1.0.1` code fixes: none.
- Release need: no, until a real bugfix is identified.

Candidate items should include:

- issue number;
- affected build;
- device and Android version;
- user-visible impact;
- privacy risk, if any;
- reproduction confidence;
- proposed fix owner;
- validation required before release.

## Privacy/security blockers

Create `v1.0.1` if any of these are confirmed:

- normal profile UI exposes server IP, host/domain, port, UUID,
  password, raw URI, base64 payload, subscription URL, or query-like
  secret label;
- diagnostics copy/export includes private profile data, subscription
  URLs, tokens, passwords, raw endpoints, or raw logs;
- imported profile data is logged, copied, or shown outside explicit
  user action;
- issue comments contain private data that needs maintainer
  intervention.

Triage rule: add `privacy-sensitive`, `profile-label`, and
`needs-triage`. Ask the reporter to edit or delete private content. Do
not quote secrets in comments, docs, or backlog.

## Connectivity blockers

Create `v1.0.1` if any of these are confirmed on `android-v1.0.0`:

- install blocker on supported Android versions;
- VPN permission allow path cannot connect;
- VPN permission cancel path leaves the UI stuck or creates fake
  connected state;
- disconnect does not close the tunnel;
- reconnect consistently fails after a successful first connection;
- app restart while connected creates misleading state;
- crash/ANR in connection, import, profile management, or diagnostics
  paths;
- broad connectivity failure across more than one device/network.

Triage labels:

- `connectivity`;
- `vpn-permission`;
- `profile-import`;
- `dns`;
- `udp`;
- `ipv6`.

## UX polish

Consider `v1.0.1` only if the UX issue blocks normal tester flow or
causes repeated incorrect reports.

Examples:

- unclear active profile selection;
- confusing duplicate import result;
- rename/delete confirmation copy is ambiguous;
- no-profile state is hard to recover from;
- error message is not persistent enough;
- diagnostics copy/export affordance is unclear.

## Diagnostics hardening

Before any `v1.0.1` release, review whether diagnostics need stronger
redaction tests for:

- UUIDs;
- VLESS/VMess/Trojan/Shadowsocks URIs;
- subscription URLs;
- `password`, `token`, `pbk`, `sid`, `spx`;
- `Authorization`, `Cookie`, `X-Api-Key`;
- raw IP addresses and hostnames in user-provided material;
- raw logcat fragments.

Raw diagnostics, screenshots, `.local/`, APK/AAB, private configs, and
VPN profiles must stay out of git.

## Network validation follow-up

`v1.0.1` does not require unrestricted production network validation,
but new network claims require evidence.

Current status:

- DNS: `pass` for the tested device/network only.
- UDP: `pass_limited`; Android-side functional evidence exists, but no
  formal threshold/outlier policy is approved.
- IPv6: `not_tested`; checked networks had no clean external IPv6
  baseline.

Follow-up candidates:

- repeat DNS leak audit on another device/network;
- define UDP loss/jitter threshold and outlier policy;
- rerun controlled Android-side UDP/iperf after threshold approval;
- test IPv6 on a network with real external IPv6;
- prove IPv6 tunnels or fails closed with no raw IPv6 fallback.

## Release criteria for v1.0.1

`v1.0.1` may be considered only if at least one real bugfix exists. Do
not release `v1.0.1` only for docs.

Release is justified by:

- privacy/security bug;
- crash/ANR;
- install blocker;
- mass connectivity problem;
- important UX fix that unblocks tester workflow;
- diagnostics redaction fix.

Required before any `v1.0.1` tag or release:

- version bump;
- unit tests for changed behavior where practical;
- Gradle unit tests and release build checks;
- signed workflow from the exact release source SHA;
- APK checksum verification;
- APK signature verification;
- AAB verification if produced;
- 16 KB native alignment verification;
- `zipalign -P 16`;
- physical smoke on a real device;
- release notes that disclose remaining UDP/IPv6 limitations;
- explicit tag/release approval.
